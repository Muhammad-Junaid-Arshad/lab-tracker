Complete Setup Guide
1

Get your Dropbox Access Token

Go to dropbox.com/developers/apps

  • Click Create app
  • Choose Scoped access → Full Dropbox
  • Give it any name (e.g. "Lab Tracker")
  • Go to Permissions tab → enable files.content.write and files.content.read
  • Go to Settings tab → scroll to OAuth 2 → click Generate access token
  • Copy the token — it starts with sl.
⚠ Keep this token private. It gives write access to your Dropbox. Do not share the HTML file publicly with the token inside it — see Step 3 for how to handle this.
2

Add token to index.html

Open index.html in a text editor. Find the CONFIG block near the top and replace:

  • dropboxToken: "YOUR_DROPBOX_ACCESS_TOKEN_HERE" → paste your token
  • That's it — save the file
3

Host on GitHub Pages (free)

Go to github.com → create a free account if needed.

  • Click New repository → name it (e.g. lab-tracker)
  • Set it to Private ← important for token safety
  • Upload both index.html and qr-codes.html
  • Go to Settings → Pages → Source: Deploy from branch → main
  • Wait ~1 minute → your URL appears: https://username.github.io/lab-tracker
✓ Private repos on GitHub Pages are accessible by URL but not indexed — your token is protected as long as you don't share the URL widely. For extra security, consider using a Dropbox app with folder-scoped access only.
4

Update QR codes with your URL

Paste your GitHub Pages URL into the box at the top of this page → click Update QR Codes → the codes regenerate instantly. Then click Print QR Codes.

5

Print & attach to devices

Click the Print QR Codes button at the top. The page prints in a clean 2-column layout — cut out each label and laminate or use a label pouch. Stick to the device.

  • QR code size is ~3×3 cm when printed — readable by any phone camera
  • Scanning opens the form directly in the browser — no app needed
  • The device name and ID auto-fill on the form
6

Add a new device

Open both index.html and qr-codes.html. In each file, find the devices array and add one entry:

  • id: "EQ-004" — unique ID
  • name: "Full Device Name"
  • short: "Short Name"
  • In index.html only, also add: csvFile: "/Lab Equipment Tracker/eq-004-name.csv"

Save → re-upload both files to GitHub → a new QR code appears automatically on this page.

7

View your data

Each device has its own .csv file inside the /Lab Equipment Tracker/ folder in your Dropbox. Open any of them directly in:

  • Excel — File → Open → Dropbox → select file
  • Google Sheets — File → Import → upload the CSV
  • Dropbox Paper — preview in browser

Photos from issue reports are saved in /Lab Equipment Tracker/photos/.
